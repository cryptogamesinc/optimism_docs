# アプリ開発者概要

OP Mainnetでの開発を検討している開発者の方へ、ここが正しい場所です。
Optimism Docsのこのエリアでは、OP Mainnetアプリケーションの構築に必要なすべての情報を見つけることができます。

## はじめに

OP Mainnetが初めての方は、[基本的なコントラクトのデプロイ](/chain/getting-started)に関するガイドから始めてみてください。
これにより、ネットワークへのコントラクトデプロイに必要な基本的なステップに慣れることができます。
OP Mainnetは[EVMと等価](https://web.archive.org/web/20231127160757/https://medium.com/ethereum-optimism/introducing-evm-equivalence-5c2021deb306)ですので、既存のEthereumスマートコントラクトスキルがOP Mainnetにも活用できることを安心してください。
ただし、EthereumとOP Mainnetの[いくつかの小さな違い](/chain/differences)に注意してください。

また、[OPネットワークでのテストガイド](/chain/testing/testing-apps)や[ローカル開発環境の実行](/chain/testing/dev-node)についてのチュートリアルも確認して、OP Mainnetデプロイメントに完全な自信を持てるようにしましょう。

## ブリッジとメッセージング

OP MainnetとEthereum間でETH、トークン、またはデータを送信するアプリケーションを構築しますか？
このドキュメントのエリアには、役立つガイドやチュートリアルがあります。
例えば、EthereumからOP Mainnet（またはその逆）へトークンをブリッジする方法を学びたい場合は、[標準トークンブリッジ](/builders/app-developers/bridging/standard-bridge)をチェックしてください。

もっと高度なものを探している場合は、[L1とL2間でデータを送信](/builders/app-developers/bridging/messaging)するガイドを見てみましょう。
あるチェーンのコントラクトが他のチェーンのコントラクト機能をトリガーすることができます。とてもクールです！
OP Mainnetの標準トークンブリッジも、この同じメッセージパッシングインフラストラクチャを内部で使用しています。
