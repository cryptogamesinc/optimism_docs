# デザイン哲学 & デザイン原則

このガイドでは、OP Stackに貢献するためのデザイン哲学とデザイン原則について説明します。

## デザイン哲学

OP Stackは、**シンプルさ**、**実用主義**、**持続可能性**、そしてもちろん**Optimism**という4つの主要な柱に基づく強力なデザイン哲学に従って構築されています。
これらの柱を理解することは重要であり、それらはOptimism全体のデザインに大きく影響を与えます。

### シンプルさ

Optimismは、提供する機能セットに対して可能な限りシンプルに設計されています。
理想的には、Optimismは安全でスケーラブルで柔軟なL2システムに必要な最小限の可動部分で構成されるべきです。
このシンプルさは、他のより複雑なL2構造に対して、Optimismのデザインにいくつかの重要な利点をもたらします。

シンプルさはエンジニアリングの負担を減らし、その結果、新しい機能の開発に時間を費やすことができます。
Optimismは可能な限り既存の実績のあるEthereumのコードとインフラストラクチャを使用することを好みます。
この哲学を実践する最も顕著な例は、OptimismのクライアントソフトウェアとしてGethを使用する選択です。

重要なインフラストラクチャを扱う場合、シンプルさはセキュリティでもあります。
私たちが書くコードの一行一行が、意図しないバグを導入する機会です。
シンプルなプロトコルは、書くべきコードが少なく、その結果、潜在的なミスの余地が少なくなります。
クリーンで最小限のコードベースは、外部の貢献者や監査人にもアクセスしやすくなります。
これらすべてが、Optimismプロトコルのセキュリティと正確性を最大化することに寄与します。

シンプルさはまた、Optimismの長期的なビジョンにとっても重要です。
Ethereumのツール群の上に私たちが書くコードの量を制限することで、既存のコードベースを直接扱うことに時間を費やすことができます。
Optimismに注がれるエンジニアリングの努力は、Ethereumにも直接利益をもたらすことができ、その逆もまた然りです。
これは、Optimismプロトコルが固まり、既存のリソースをコアなEthereumインフラストラクチャに再配分できるようになるにつれて、さらに顕著になるでしょう。

### 実用主義

その理想主義にもかかわらず、Optimismの背後にあるデザインプロセスは最終的には実用主義によって駆動されています。
コアOptimismチームには現実世界の制約があり、Optimism上で構築するプロジェクトには現実世界のニーズがあり、Optimismと関わるユーザーには現実世界の問題があります。
Optimismのデザイン哲学は、理論的な完璧さよりもユーザーと開発者のニーズを優先します。
時には、最良の解決策が最も美しいものではないこともあります。

Optimismは、コアチームが限られた専門分野を持つことを理解して開発されています。
Optimismは反復的に開発され、ユーザーからのフィードバックを継続的に取り入れるよう努めています。
今日の多くのコアOptimism機能（例えば[EVM Equivalence](https://web.archive.org/web/20231127160757/https://medium.com/ethereum-optimism/introducing-evm-equivalence-5c2021deb306)）は、この反復的なプロトコル開発アプローチによってのみ可能になりました。

### 持続可能性

Optimismは長期的な視点で取り組んでいます。
アプリケーション開発者は、自分たちが構築しているプラットフォームが長期にわたって運用され、競争力を維持することを保証する必要があります。
Optimismのデザインプロセスは、長期的な持続可能性とスケーラビリティへの近道を避けるというアイデアに基づいて構築されています。
最終的には、スケーラブルなシステムは、それを支えるエコシステムなしでは何の意味もありません。

持続可能性は、私たちのシンプルさの哲学と密接に関連しながら、Optimismのプロトコルデザインに積極的に影響を与えています。
コードベースが複雑であればあるほど、コア開発チーム外の人々が積極的に貢献することが難しくなります。
私たちのコードベースをシンプルに保つことで、プロトコルを長期的に維持するのを助ける貢献者のコミュニティをより大きく構築することができます。

### Optimism

もちろん、これらすべてはOptimismの精神なしには可能ではありません。
Ethereumのビジョンに対する私たちの楽観主義が、このプロジェクトを前進させ続けています。
私たちはEthereumの楽観的な未来、つまり私たちの生活を調整する機関との関係を再設計できる未来を信じています。

Optimismは単独のブロックチェーンのように見えますが、最終的にはEthereumへの拡張として設計されています。
新しい機能を作成したり、既存のものを簡素化したりするときは常にこれを念頭に置いています。
Optimismは、実用的な理由だけでなく、OptimismがEthereumの成功のために存在するため、可能な限りEthereumに近づけています。
私たちは、Optimismのデザインを見るときに、この哲学の影響を感じていただけることを願っています。

## USEful Softwareのためのデザイン原則

OP StackはUSEfulなソフトウェアです。OP Stackは、Optimism CollectiveによってOptimismを動かすために構築された、L2ブロックチェーンエコシステムを構築するための一連のソフトウェアコンポーネントです。
OP Stackに追加されるコンポーネントは、**U**tility（有用性）、**S**implicity（シンプルさ）、**E**xtensibility（拡張性）の3つの重要なデザイン原則に従って構築されるべきです。
これらの原則に従うソフトウェアは、Optimism Collectiveにとって**USE**fulなソフトウェアです！

### Utility（有用性）

OP Stackの一部であるためには、それがOptimism Collectiveを動かすのに役立つべきです。
この条件は、スタックに含まれるソフトウェアの種類を導きます。
例えば、ユーザーが[Superchain](https://app.optimism.io/superchain/)をより簡単に調べることができる強力なオープンソースのブロックエクスプローラーは、OP Stackへの素晴らしい追加となるでしょう。

有用性はOP Stackへの包含にとって重要ですが、実験することを恐れるべきではありません。
クレイジーなことをしてみてください。
明確な有用性がなくても、これまでに構築されたことのないものを作ってください。絵文字のためのブロックチェーンを作るとか、何でもいいです。楽しんでください！

### Simplicity（シンプルさ）

複雑なコードはスケールしません。
OP Stackに含まれるコードはシンプルであるべきです。

シンプルさはエンジニアリングの負担を減らし、その結果、Collectiveは既存のものを再構築するのではなく、新しい機能の開発に時間を費やすことができます。
OP Stackは、可能な限り既存の実績のあるコードとインフラストラクチャを使用することを好みます。
この哲学を実践する最も顕著な例は、OP Stackのデフォルトの実行エンジンとしてGethを使用する選択です。

重要なインフラストラクチャを扱う場合、シンプルさはセキュリティとメンテナンス性でもあります。
書かれたコードの一行一行が、バグや脆弱性を導入する機会です。
シンプルなプロトコルは、書くべきコードが少なく、その結果、潜在的なミスの余地が少なくなります。
クリーンで最小限のコードベースは、外部の貢献者や監査人にもアクセスしやすくなります。
これらすべてが、OP Stackのセキュリティと正確性を最大化することに寄与します。

### Extensibility（拡張性）

良いOP Stackのコードは、本質的にオープンで協力的で拡張可能です。
協力は、孤立した開発から抜け出すことを可能にします。
協力は、同じコンポーネントを何度も再構築するのではなく、お互いの仕事の上に構築するためにより多くの時間を費やすことを可能にします。
協力は、私たちが一緒に勝つ方法です。

拡張可能なコードは、他の人がそのコードを使って、またその上に構築したいと考えるという心構えで設計されるべきです。
実際には、これはコードがオープンソース（寛容なライセンスの下で）、クリーンなAPIを公開し、一般的にモジュール化されていて、他の開発者が比較的容易にコードの機能を拡張できることを意味します。
拡張性は、Optimism Collectiveエコシステム内での協力のスーパーパワーを解き放つ重要なデザイン原則です。

## OP Stackへの貢献

OP Stackは、誰でも貢献できる分散型のソフトウェアスタックです。
OP Stackへの貢献に興味がある場合は、[コミュニティページのContributingセクション](https://community.optimism.io)をチェックしてください。
もちろん、Optimism Collectiveに影響を与えるソフトウェアは、[Retroactive Public Goods Funding](https://app.optimism.io/retropgf)を受け取る資格があります。
OP Stackのために構築し、素晴らしいオープンソースソフトウェアを書くことで報酬を得ましょう。これ以上の喜びはありません！
