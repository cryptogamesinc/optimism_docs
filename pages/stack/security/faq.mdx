# OP Stack Security FAQs

<Callout type="warning">
  🚧 OP Stackは進行中の作業です。OP Stackの全体的なセキュリティと分散化を改善するために常に努力しています。
</Callout>

## 分散型コンテキストにおけるセキュリティ

OP StackはOptimismを支える分散型開発スタックです。OP Stackのコンポーネントは、Optimism Collective内のさまざまなチームによって維持される場合があります。一般的には、スタック自体のセキュリティモデルよりも、OP Stackを基にした特定のチェーンのセキュリティモデルについて話す方が簡単です。**OP Stackのセキュリティの基本は、安全なデフォルトを作成しつつ、開発者にスタックを変更・拡張する柔軟性を提供することです。**

## FAQ

### 全てのOP Stackチェーンは安全ですか？

OP Stackベースのブロックチェーンのセキュリティモデルは、そのコンポーネントに使用されるモジュールに依存します。OP Stackの柔軟性と許可のない性質のため、誰かが悪意を持って、または誤って、コアセキュリティ機能を使用せずにOP Stackの他のコンポーネントを使用するチェーンを設定することは常に可能です。**OP Stackの目標は、安全なデフォルトを提供することです。**

また、他のシステムと同様に、**OP Stack（またはOP Stackを使用して構築されたチェーン）には、ETHやトークンの一部または全部の喪失につながる可能性のある未知のバグが含まれている場合があります。**
OP Stackのコードベースの多くのコンポーネントは[監査](https://github.com/ethereum-optimism/optimism/tree/v1.1.4/technical-documents/security-reviews)されていますが、成功した監査が新興技術のすべての潜在的なリスクを排除するわけではなく、完了した監査がコードベースが完全にバグがないことを意味するわけではありません。
OP Stackを使用することは、OP Stackのコードベース内のバグのリスクにさらされることを理解することが重要です。

### OP Stackを変更しても安全ですか？

何事もそうですが、自己責任でOP Stackを変更してください。スタックの変更が安全であるという保証はありません。完全に自分が何をしているか分からない場合は、OP Stackが提供するより安全なデフォルトを使用してください。現時点では、OP Stackは特に変更に対して柔軟ではなく、**スタックの標準Rollup構成外の変更についての技術サポートを期待すべきではありません。**

### フォルトプルーフを使用できますか？

**まだです。** OP Stackには現在フォルトプルーフシステムがありません。**システムが7日間のチャレンジウィンドウ内でアップグレード可能な場合（「高速アップグレードキー」）、フォルトプルーフはシステムのセキュリティを意味のある形で向上させません。** 高速アップグレードキーを持つシステムは、セキュリティに関して完全にアップグレードキーに依存します。

フォルトプルーフは重要なマイルストーンであり、OP Stackの最優先事項です。その間、OP Stackは、フォルトプルーフが本番環境で利用可能になる前にセキュリティを向上させたいシステムのために、他のいくつかの優れたセキュリティオプションを提供することができます。

### OP Stackのセキュリティを向上させるためにどうすればいいですか？

OP Stackのセキュリティを向上させる最も簡単な方法の一つは、バグや脆弱性を探すことです。[OP Mainnet, OP Stackのユーザーは、最大のバグバウンティの一つ（史上最高）を持っています。](https://immunefi.com/bounty/optimism/)。OP Mainnetのコードベース（ひいてはOP Stack）の重大なバグを発見することで、最大で$2,000,042を稼ぐことができます。

OP Stackは分散型開発スタックであることを忘れないでください。誰でも[スタックの設計原則](/stack/protocol/design-principles)に従ってソフトウェアを構築することで、OP Stackに貢献を始めることができます。OP Stackのユーザーが利用できるようなコンポーネント、例えば代替クライアントやプルーフ実装などを構築することで、常にOP Stackのセキュリティを向上させることができます。

### バグをどこに報告すればいいですか？

脆弱性の報告と利用可能なバグバウンティプログラムの詳細については、[セキュリティポリシー](/chain/security/security-policy)を参照してください。
